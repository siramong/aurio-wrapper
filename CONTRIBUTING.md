# Contributing to Aurio SDK

Thank you for contributing to Aurio SDK! This document provides guidelines for development and contributions.

## Development Setup

```bash
# Clone the repository
git clone https://github.com/yourusername/aurio-sdk.git
cd aurio-sdk

# Install dependencies
npm install

# Build the SDK
npm run build

# Type check
npm run typecheck
```

## Project Structure

```
src/
  core/              ← Solana + SPL token logic (NO React/React Native)
    aurio.ts
    constants/
      web3.ts
    index.ts
  hooks/             ← Optional React hooks
    useAurio.ts
    index.ts
  index.ts           ← Main SDK entrypoint

examples/
  react-native/      ← React Native/Expo example components
    *.tsx

dist/                ← Compiled output (generated by npm run build)
```

## Architecture Principles

### 1. Core Module (src/core/)
- **Zero React dependencies** - Works in Node.js, browsers, React Native
- **Pure blockchain logic** - Solana web3.js + SPL token operations
- **Async-first** - All balance and transaction operations are Promise-based
- **Error handling** - Proper error messages and validation

### 2. Hooks Module (src/hooks/)
- **Optional** - Not required to use the SDK
- **React only** - Uses React hooks (useState, useEffect)
- **Wraps core** - All hooks use core functions internally
- **React Native compatible** - Works with React Native without native modules

### 3. Examples (examples/)
- **Not part of SDK** - Zero runtime impact on published package
- **Documentation** - Show real-world usage patterns
- **React Native** - Demonstrates hooks and core API usage

## Adding Features

### Adding to Core SDK

1. Add type definitions to `src/core/aurio.ts`:
```typescript
export interface MyNewParams {
  wallet: string;
  // ...
}

export async function myNewFunction(params: MyNewParams): Promise<Result> {
  // implementation
}
```

2. Export from `src/core/index.ts`:
```typescript
export * from "./aurio";
```

3. Test with TypeScript:
```bash
npm run typecheck
```

4. Build:
```bash
npm run build
```

### Adding a React Hook

1. Create hook in `src/hooks/useAurio.ts` or new file:
```typescript
import { useState, useEffect } from "react";
import { myNewFunction } from "../core";

export function useMyFeature(wallet: string) {
  const [state, setState] = useState(null);
  
  useEffect(() => {
    myNewFunction({ wallet }).then(setState);
  }, [wallet]);
  
  return state;
}
```

2. Export from `src/hooks/index.ts`

3. Ensure React and React Native are listed as `peerDependencies` in package.json

## Code Style

- **TypeScript**: Strict mode enabled - no `any` types
- **Naming**: camelCase for functions/variables, PascalCase for types/interfaces
- **Exports**: Prefer named exports, only use default export for the main entrypoint
- **Comments**: Add JSDoc comments for public APIs
- **Async**: Use async/await, avoid Promise chains in most cases

## Testing

Currently, manual testing is recommended:

```typescript
// test.ts
import { getAurioBalance } from "./dist/index.js";

const balance = await getAurioBalance("wallet-address");
console.log(balance);
```

## Git Workflow

1. Create a feature branch:
```bash
git checkout -b feat/my-feature
```

2. Commit with clear messages:
```bash
git commit -m "feat: add new balance query function"
git commit -m "fix: handle edge case in decimals"
git commit -m "chore: update dependencies"
```

3. Push and create a Pull Request:
```bash
git push origin feat/my-feature
```

## Commit Message Format

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Types
- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation changes
- `style`: Code style changes (formatting, semicolons, etc.)
- `refactor`: Code refactoring without feature changes
- `perf`: Performance improvements
- `chore`: Build, dependencies, or tooling changes
- `test`: Adding or updating tests

### Scopes (optional)
- `core`: Core SDK logic
- `hooks`: React hooks
- `constants`: Constants and config
- `docs`: Documentation

### Examples
```
feat(core): add getAurioTokenSupply function
fix(core): handle missing token accounts gracefully
docs: add usage examples to README
chore(deps): update @solana/web3.js to latest
```

## Before Submitting a PR

- [ ] Run `npm run typecheck` - no TypeScript errors
- [ ] Run `npm run build` - build succeeds
- [ ] Update README.md if adding public APIs
- [ ] Add example usage if adding new features
- [ ] Test with your use case
